# Task: Fix Liquid Glass Pill Effect in macOS Overlay

## Goal
The `TranscriptionPill` (an NSView) needs to blur/refract the animated particle dots behind it, creating a frosted glass effect. Currently the dots are visible through the pill un-blurred, or the blur captures the wrong content.

## Design Reference
- Figma: https://www.figma.com/design/IGjwpE5m7vLaCThHgoy0cw/Mentistac?node-id=197-13273
- The pill should look like frosted glass ‚Äî the particle dots behind it should be blurred into a smooth warm color wash, not visible as individual dots
- Figma specifies: `Glass: Effect(type: GLASS, radius: 8)` ‚Äî a simple backdrop blur
- The glass should be semi-transparent with warm particle colors bleeding through
- Subtle white border stroke around the pill edge

## Architecture
Everything lives in a single transparent full-screen `NSPanel` (the overlay):

```
OverlayPanel (NSPanel, transparent, click-through, level: .screenSaver)
‚îî‚îÄ‚îÄ contentView (NSView, wantsLayer=true, clear background)
    ‚îú‚îÄ‚îÄ contentView.layer
    ‚îÇ   ‚îú‚îÄ‚îÄ gradientLayer (CAGradientLayer) ‚Äî full-screen dark gradient, transparent‚Üí75% black
    ‚îÇ   ‚îî‚îÄ‚îÄ particleLayer (CALayer) ‚Äî animated dots rendered as CGImage at 30fps
    ‚îî‚îÄ‚îÄ pill (TranscriptionPill, NSView subview) ‚Äî positioned bottom-center
```

The particles are rendered into a reusable `CGContext` every frame and set as `particleLayer.contents` (a CGImage). The particle layer covers the bottom 44% of the screen.

The pill sits on top of the particles as an NSView subview of the same contentView.

## What Has Been Tried

### 1. NSVisualEffectView with `.behindWindow` blending
- Blurs the desktop/apps BEHIND the overlay window, NOT the particles (which are in the same window)
- Result: no particle refraction at all

### 2. NSVisualEffectView with `.withinWindow` blending
- Should blur content within the same window
- Tried materials: `.popover`, `.sheet`, `.underWindowBackground`, `.hudWindow`
- Problem: all materials add heavy built-in tinting that makes the pill too dark/opaque
- The blur does seem to partially work but the material dominates the appearance
- Best result was `.popover` at `alphaValue = 0.65` but still too dark and particles barely show through

### 3. CALayer.backgroundFilters with CIGaussianBlur
- Applied `CIFilter(name: "CIGaussianBlur")` as a backgroundFilter on the pill's glass layer
- Result: no blur at all ‚Äî the dots are fully crisp through the pill
- Likely because backgroundFilters doesn't capture sibling CALayer content in an NSView layer tree

## Current Code

The file to modify is `Sources/relay-runner/Overlay/TranscriptionPill.swift`. The current implementation uses approach #2 (NSVisualEffectView `.withinWindow`).

Key constraints:
- macOS 14+ target (Swift Package Manager, `platforms: [.macOS(.v14)]`)
- The pill must remain an NSView (it has NSTextField subviews for text labels)
- The particles are CALayer content (CGImage) in the same window, not in a separate window
- The overlay panel is transparent, borderless, click-through
- Performance matters ‚Äî the particle renderer runs at 30fps

## Ideas to Explore
- Rendering a blurred copy of the particle region behind the pill directly in the particle renderer (passing pill frame to the renderer)
- Using a separate lower NSWindow for the particles so `.behindWindow` captures them
- Custom CALayer compositing with `CIFilter` applied to a snapshot
- Using `NSView.bitmapImageRepForCachingDisplay(in:)` to snapshot and blur the particles
- Any other macOS technique to blur in-window CALayer content behind an NSView
